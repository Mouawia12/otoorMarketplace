Improve mobile responsiveness across the app (no backend changes).

Scope:
1) Global breakpoints & layout
- Use responsive breakpoints: sm <640, md <768, lg <1024, xl <1280.
- Apply container padding: px-4 on mobile, px-6 md:px-8 lg:px-12 on larger screens.
- Ensure section spacing follows 12/16/24/32px rhythm per breakpoint.

2) Header / Navbar
- Convert desktop nav to a collapsible mobile header:
  - Hamburger menu → slide-in drawer (left for EN, right for AR).
  - Keep brand logo centered; search icon toggles an overlay search on mobile.
- Increase touch targets to 44x44px min; add focus-visible styles.
- Ensure language switch and auth buttons fit in a single row on small screens.

3) Footer
- Stack columns vertically on mobile (accordion optional).
- Ensure all Footer <Link to="..."> navigate client-side (no full reloads).
- Add comfortable padding-block (16–20px) and tighter gaps on small screens.

4) Catalog grids (/new, /used, /auctions)
- Grid columns: 1 col on <480px, 2 cols on >=480px, 3 on md, 4 on lg.
- ProductCard: image aspect-ratio 3:4; clamp title to 2 lines; price visible; buttons wrap neatly.
- Lazy-load images and add min tap targets.

5) Product detail (/p/:id)
- Move gallery thumbnails to a horizontal scroll under the main image on mobile.
- Sticky bottom action bar: [Buy Now] [Favorite] (safe-area aware).
- Ensure long titles wrap without layout shift.

6) Auction page (/auction/:id)
- Countdown visible and legible on small screens.
- Bid input + +/- and Place Bid in a single sticky bottom bar.
- Bid log: make table horizontally scrollable with sticky header on mobile.

7) Buyer/Seller/Admin tables
- Enable horizontal scroll for wide tables.
- Reduce padding and font-size slightly on mobile; keep row height >= 44px.
- Add compact mode toggle for dense data.

8) Forms & modals
- Inputs full-width on mobile; label above input.
- Buttons stack vertically with 8–12px gap.
- Modal content max-width 90vw; prevent off-screen overflow.

9) Typography & icons
- Base font-size 16px mobile; scale up on md+.
- Use line-clamp utilities to prevent overflow.
- Ensure icons don’t shrink below 20–24px on mobile.

10) Performance
- Defer non-critical scripts; lazy-load heavy routes.
- Use responsive images (sizes/srcSet) where applicable.

11) RTL/LTR
- All spacing uses logical properties (padding-inline, margin-inline, inset-inline).
- Drawer opens from inline-end for RTL and inline-start for LTR.
- Verify direction switches instantly with language toggle.

Acceptance:
- No horizontal scrolling on main pages except intended data tables.
- Header, footer, catalog cards, product, and auction screens look clean on 360–414px widths.
- All tap targets >= 44px; no layout jumps during language toggle.
- Minimal console warnings.
- Create ONE checkpoint and PRINT CHANGED PATHS ONLY.

Constraints:
- Touch ONLY layout, CSS, and component structure. No API/business logic changes.
- Keep output minimal.
